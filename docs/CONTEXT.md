# openshoki

メニューバー／タスクバーに常駐して音声を録音する、Rust 製のデスクトップアプリ。
ウィンドウを主役にせず、常駐したまま素早く録音を開始・停止できることを狙う。

## 概要

- 形態: デスクトップアプリ（メニューバー／タスクバー常駐型）
- 言語/技術スタック: Rust。GUI は **Slint**、トレイ常駐は **tray-icon** クレート。録音まわりの
  クレート選定は未確定（録音機能のプランで決める）。
- 実行/利用方法: `cargo run` で起動し、メニューバー／タスクバーのアイコンから操作する想定。
  起動時はウィンドウを出さずトレイに常駐する。

## 構成

- 想定するプロジェクト構成（基盤プランで確定。`docs/plans/20260627-app-foundation.md`）:
  - `Cargo.toml` — クレート定義・依存（`slint` / `tray-icon`、ビルド依存 `slint-build`）。
  - `build.rs` — `slint_build::compile` で `.slint` をコンパイル。
  - `ui/` — Slint の UI 定義（`app-window.slint` 等）。
  - `src/main.rs` — エントリ。トレイ初期化と Slint イベントループ起動。
  - `src/tray.rs` — トレイアイコン/メニュー構築とイベントのディスパッチ。
  - `assets/` — トレイアイコン等の素材。

## 主要な設計判断

- **常駐型を前提にする**: 通常のウィンドウアプリではなく、メニューバー／タスクバーに常駐して
  いつでも録音を始められる体験を中心に据える。UI 設計・ライフサイクルはこの前提に従う。
- **クロスプラットフォーム**: macOS のメニューバーと Windows／Linux のタスクバーの双方を
  視野に入れる。OS ごとの常駐 UI・録音 API の差異がハマりどころになりやすい。
- **GUI は Slint、トレイ常駐は tray-icon**: Slint 単体にトレイ常駐 API は無いため、
  `tray-icon` で常駐とメニューを担い、Slint でウィンドウ UI を担う。両者を winit ベースの
  Slint イベントループ上で協調させる（トレイのメニューイベントをポーリングで Slint 側へ橋渡し）。
- **録音は cpal + mp3lame-encoder で MP3 出力**: マイク取得は `cpal`（既定入力デバイス）、
  エンコードは `mp3lame-encoder`（libmp3lame）で `.mp3` に書き出す。形式は互換性重視で MP3。
- **マイク音声とシステム音声は別ファイルに保存**: 将来の文字起こしのため、マイク（発話）と
  スピーカー等のシステム音声（再生音）は混ぜず、音源ごとに別の MP3 に保存する。1 回の録音
  セッションが音源ごとの複数ファイル（`-mic` / `-system`）を生む。
- **システム音声の取得は OS 依存、macOS 先行で ScreenCaptureKit**: スピーカー出力の
  ループバック取得は共通 API が無く、macOS は `screencapturekit`（ScreenCaptureKit）で行う。
  Windows（WASAPI loopback）/ Linux（monitor source）は後続。マイク録音は全 OS 共通（`cpal`）。
- **設定は OS 標準ディレクトリに TOML で永続化**: ユーザー設定（録音ファイルの保存先など）は
  `directories` で得る OS 標準の設定ディレクトリに、`serde` + `toml` で保存・復元する。
  読み込み失敗時はデフォルトへフォールバックしてアプリは落とさない。

## Language（用語）

このプロジェクトで意味を固定して使う言葉。実装・プラン・issue・PR でこの語を使う。

**常駐**:
アプリがウィンドウを閉じても終了せず、メニューバー／タスクバーに居続けている状態。
openshoki の基本動作モード。
_避ける_: 「バックグラウンド起動」「最小化」

**録音セッション**:
録音開始から停止までの 1 回分の録音。1 つの音声ファイルに対応する単位。
_避ける_: 「レコーディング」「キャプチャ」
